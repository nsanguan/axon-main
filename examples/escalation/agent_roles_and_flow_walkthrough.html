Role Map
Flow ทีละ Step
Branch: ไม่มี Stock
Pattern C — ใครคือใคร
E
Executive Agent = ผู้จัดการใหญ่ (กองบัญชาการ)
รับ user message → classify intent → ตัดสินใจว่าจะ route ไป flow ไหน
input: "ตรวจสอบ Stock WH01..." → output: intent="stock_check_and_reorder"
1 in 1000 intent → 1 flow ไม่ทำงานเอง แค่ตัดสินใจ
S
Supervisor Node = ผู้จัดการ Flow (หัวหน้าทีม)
ควบคุมลำดับงานภายใน flow — ใครทำก่อน ใครทำหลัง ถ้าเจอ condition ให้ทำอะไร
stock_check_reorder_flow: WhereHouseAgent → [check result] → BuyerAgent
conditional_edge ใน LangGraph handover logic อยู่ที่นี่
M
Manager Agents = ผู้เชี่ยวชาญ (พนักงานมือโปร)
WhereHouseAgent — เชี่ยวชาญด้าน warehouse, stock, location
BuyerAgent — เชี่ยวชาญด้าน PO, supplier, price
แต่ละตัวมี PydanticAI contract + MCP tools ของตัวเอง
ทำงานจริง เรียก MCP ไม่รู้จัก agent อื่น
สรุป mapping กับ diagram Pattern C
Executive = node แรกของทุก flow  ·  Supervisor = conditional edges  ·  Managers = worker nodes
User: "ตรวจสอบ Stock WH01 Store KB-001 ต้องเพิ่มอะไร"
0
API Gateway
รับ message → สร้าง thread_id → ส่งเข้า LangGraph
POST /chat { message, user_id, thread_id }
↓
E
Executive Agent — Intent Classification
LLM อ่าน message → extract entities → map intent
intent: "stock_check_and_suggest_reorder"
entities: { wh: "WH01", store: "KB-001" }
route → stock_check_reorder_flow
↓
S
Supervisor — เริ่ม flow: stock_check_reorder_flow
กำหนด step แรก = WhereHouseAgent ตรวจ stock ก่อน
state: { wh="WH01", store="KB-001", stock_result=None, po_result=None }
↓
M1
WhereHouseAgent — ตรวจ Stock
เรียก MCP store tools: get_stock_onhand(wh="WH01", store="KB-001")
→ result: { onhand: 0, min_stock: 50, items_below_min: ["SKU-A", "SKU-B", "SKU-C"] }
MCP: stock_levels tool onhand = 0 → trigger condition
↓
S
Supervisor — ตรวจ Condition
stock_onhand == 0 → route ต่อไป BuyerAgent
conditional_edge: check_stock_result → "need_reorder" → buyer_agent_node
↓
M2
BuyerAgent — เสนอแผนซื้อ
เรียก MCP buyer tools: get_supplier_price(skus), suggest_po(items, qty)
→ result: { supplier: "SUP-01", items: 3, total_qty: 150, est_cost: 45000 }
MCP: price_lookup + purchase_order tools
↓
R
Response synthesis → User
สรุปผลรวม: stock status + รายการที่ต้องซื้อ + ราคา
stream กลับ WebApp ผ่าน SSE
Branch Logic — เมื่อ WhereHouseAgent พบว่า Stock = 0
M1
WhereHouseAgent ส่ง result กลับ Supervisor
stock_result = { wh: "WH01", store: "KB-001", onhand: 0, shortage: ["SKU-A","SKU-B","SKU-C"] }
Supervisor conditional_edge ตรวจ 3 กรณี
onhand > min → END (แค่รายงาน)
0 < onhand < min → WhereHouseAgent แนะนำ replenish จาก Main WH
onhand == 0 + Main WH ก็ไม่มี → BuyerAgent สั่งซื้อ ← กรณีนี้
S
Supervisor route ไป BuyerAgent
ส่ง context ครบ: ไม่มีทั้ง store และ WH หลัก
state.shortage_items = ["SKU-A","SKU-B","SKU-C"]
state.source = "external_purchase_required"
↓
M2
BuyerAgent รับงาน
PydanticAI Input schema ที่รับมา
class BuyerInput(BaseModel):
  shortage_items: list[str]
  target_store: str # KB-001
  target_wh: str # WH01
ถ้า LLM ตอบผิด schema → PydanticAI retry อัตโนมัติ
↓
MCP
MCP Buyer Tools ที่ถูกเรียก
get_supplier_list(items) → เลือก supplier ที่ดีที่สุด
get_price_quote(supplier, items) → ได้ราคา
draft_purchase_order(supplier, items, qty) → ร่าง PO
→ PO draft: SUP-01 | SKU-A×50 SKU-B×60 SKU-C×40 | 45,000 THB
↓
R
Response ที่ User เห็น
Stock WH01 / Store KB-001: ไม่มี onhand ทั้ง store และ WH หลัก
แนะนำสั่งซื้อ 3 รายการจาก SUP-01 รวม 45,000 บาท
ต้องการยืนยัน PO หรือปรับ qty ก่อนส่งไหม?
Human-in-the-loop: รอ confirm ก่อน submit PO จริง
